#关于hexo与github使用过程中的问题与笔记

一、搭建个人博客前提：

Node.js、git 环境安装

二、Hexo安装

安装Hexo

npm install hexo -cli -g #安装hexo模块
hexo init blog #通过hexo创建一个blog项目
cd blog #打开blog文件夹
npm install
hexo server #开发服务

三、使用github

GitHub中创建一个与用户名一致的仓库，以 io 为后缀，且一个账户只能建立一个。

如建立一个名叫injoinD.github.io的仓库，其中injoinD为用户名。

四、配置git信息（Hexo和GitHub之间利用SSH通信）

1.设置Git的user name和Email：

git config --global user.name "用户名"
git config --global user.email "邮箱"

2.生成密钥

ssh-keygen -t rsa -C "邮箱"

打开以.ssh结尾的文件夹，将里面的内容配置到GitHub项目里的SSH and GPG keys中

五、配置Deployment

在blog项目根目录下找到_config.yml文件，找到Deployment，做出如下修改：

deploy:
  type: git
  repo: git@github.com:yourname/yourname.github.io.git
  branch: master

如：仓库地址是git@github.com:injoinD/injoinD.github.io.git,所以配置如下：

deploy:
  type: git
  repo: git@github.com:injoinD/injoinD.github.io.git
  branch: master

六、安装hexo-diployer-git自动部署发布工具

npm install hexo-deployer-git --save

七、生成静态文件部署到GitHub

通过hexo命令，发布到GitHub上

hexo g #生成网站静态文件到默认设置的 public 文件夹
hexo d #自动生成网站静态文件，并部署到设定的仓库
hexo clean #清除缓存文件 db.json 和已生成的静态文件 public

通过链接 injoinD.github.io即可访问 injoinD为自己的用户名
