1.Git的优势

  • 大部分操作在本地完成，不需要联网
  • 完整性保证
  • 尽可能添加数据而不是删除或修改数据
  • 分支操作非常快捷流畅
  • 与 Linux 命令全面兼容

3.分支管理

  • 3.1 什么是分支？ 

    在版本控制过程中，使用多条线同时推进多个任务。

  • 3.2 分支的好处？

    • 同时并行推进多个功能开发，提高开发效率
    • 各个分支在开发过程中，如果某一个分支开发失败，不会对其他分支有任 何影响。失败的分支删除重新开始即可。

  • 3.3 分支操作

    • 创建分支 git branch [分支名]

    • 查看分支 git branch -v

    • 切换分支 git checkout [分支名]

    • 合并分支

      • 第一步：切换到接受修改的分支（被合并，增加新内容）上

        git checkout [被合并分支名]

      • 第二步：执行 merge 命令 

        `git merge [有新内容分支名]`

  • 3.4 冲突的解决

    • 第一步：编辑文件，删除特殊符号

    • 第二步：把文件修改到满意的程度，保存退出

    • 第三步：git add [文件名]

    • 第四步：git commit -m "日志信息"

      注意：此时 commit 一定不能带具体文件名
